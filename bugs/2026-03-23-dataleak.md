# Data Leak & Security Audit — 2026-03-23

## Scope
`server/index.js`, `server/proxy.js`, `server/ai/trainer.js`, `server/ai/model.js`, `server/ai/buffer.js`, `config.js`, `client/src/`

---

### SEC-001: API `/api/ai/predict` zwraca surowe dane modelu
- **Risk:** średni
- **Lokalizacja:** `server/index.js:62` (res.json(result))
- **Opis:** Endpoint `/api/ai/predict` zwraca pełny obiekt z `predict()`, który zawiera raw policy array, value scalar, selected move index. Każdy kto zna adres może wysłać POST z dowolną planszą i otrzymać wewnętrzne dane modelu AI. Nie ma autentykacji ani rate limitingu.
- **Propozycja naprawy:** Zwracać tylko `{ move: result.move }` — wewnętrzne `policy` i `value` nie powinny być publiczne.

### SEC-002: API `/api/ai/params` zwraca pełny status wewnętrzny
- **Risk:** średni
- **Lokalizacja:** `server/index.js:88` (res.json({ ok: true, ...trainer.getStatus() }))
- **Opis:** `trainer.getStatus()` zwraca `modelParams`, `bufferSize`, `avgRoundTimeMs`, `last10Times`, `totalTimeMs` — wewnętrzne metryki treningowe niepotrzebne klientowi.
- **Propozycja naprawy:** Zwracać tylko `{ ok: true }` lub okrojony status.

### SEC-003: CORS origin — potencjalnie za otwarty
- **Risk:** niski (konfiguracja)
- **Lokalizacja:** `config.js:13`, `server/index.js:22`
- **Opis:** `corsOrigin: process.env.CORS_ORIGIN || 'http://localhost:3000'`. Jeśli w produkcji `CORS_ORIGIN=*`, Socket.IO zaakceptuje połączenia z dowolnej domeny. Brak walidacji wartości zmiennej środowiskowej.
- **Propozycja naprawy:** Dodać walidację — odrzucać `*` w produkcji, lub logować ostrzeżenie.

### SEC-004: Brak nagłówków bezpieczeństwa HTTP
- **Risk:** niski
- **Lokalizacja:** `server/index.js` (brak middleware)
- **Opis:** Brak `X-Content-Type-Options`, `X-Frame-Options`, `Strict-Transport-Security`, `Content-Security-Policy`. Express nie ustawia ich domyślnie.
- **Propozycja naprawy:** Dodać `helmet` middleware lub ręcznie ustawić nagłówki.

### SEC-005: Brak limitu rozmiaru body requestu
- **Risk:** niski (DoS)
- **Lokalizacja:** `server/index.js:26` (app.use(express.json()))
- **Opis:** `express.json()` bez limitu — atakujący może wysłać ogromny JSON, zużywając pamięć serwera.
- **Propozycja naprawy:** `app.use(express.json({ limit: '1mb' }))`

### SEC-006: WebSocket broadcast — stan gry i metryki dla wszystkich
- **Risk:** niski (aplikacyjny)
- **Lokalizacja:** `server/index.js:75,93,251,258`, `server/ai/trainer.js` (io.emit)
- **Opis:** `io.emit('train', { loss })`, `io.emit('paramsChange', ...)`, `io.emit('selfPlayStatus', ...)` — wszystkie metryki treningowe lądują u każdego podłączonego klienta. W kontekście gry warcabowej to akceptowalne (plansza jest publiczna), ale `loss` i parametry modelu mogą być uznane za wrażliwe.
- **Propozycja naprawy:** Niski priorytet — dla aplikacji do nauki to OK. W produkcji rozważyć `socket.emit` zamiast `io.emit` dla metryk.

### SEC-007: console.error loguje pełne obiekty błędów (stack trace)
- **Risk:** niski
- **Lokalizacja:** `server/index.js:61,78,105,164`
- **Opis:** `console.error('[AI] Predict error:', err)` — `err` to pełny obiekt Error ze stack trace, który zawiera ścieżki plików na serwerze. W środowisku produkcyjnym logi mogą być dostępne publicznie.
- **Propozycja naprawy:** `console.error('[AI] Predict error:', err.message)` — logować tylko message, nie cały obiekt.

### SEC-008: WebSocket `setParams` broadcastuje wewnętrzne modelParams
- **Risk:** niski
- **Lokalizacja:** `server/index.js:431` (io.emit('paramsUpdate', { modelParams: { ...trainer.modelParams } }))
- **Opis:** Po zmianie parametrów, `modelParams` (layers, neurons, lr, batchSize, dropout, activation) są broadcastowane do WSZYSTKICH klientów przez `io.emit`. Każdy obserwator widzi pełną architekturę sieci.
- **Propozycja naprawy:** Broadcastować tylko `{ ok: true }` lub minimalne info. Pełne params wysyłać tylko do klienta który je zmienił.

### SEC-009: Brak walidacji formatu tablicy `board` w `/api/ai/predict`
- **Risk:** niski
- **Lokalizacja:** `server/index.js:49-50`
- **Opis:** Walidacja sprawdza tylko `if (!board || !legalMoves)` — nie sprawdza czy `board` jest tablicą, czy ma prawidłowy rozmiar, czy wartości są numeryczne. Złośliwy payload może spowodować nieoczekiwane zachowanie modelu.
- **Propozycja naprawy:** Dodać: `if (!Array.isArray(board) || board.length !== 64)` → 400.

### SEC-010: process.env.TF_ENABLE_ONEDNN_OPTS na szczycie pliku
- **Risk:** niski (informacyjny)
- **Lokalizacja:** `server/index.js:1`
- **Opis:** `process.env.TF_ENABLE_ONEDNN_OPTS = process.env.TF_ENABLE_ONEDNN_OPTS || '1'` — ustawia zmienną środowiskową TF. Nie jest to wyciek, ale powinno być w osobnym module konfiguracyjnym.
- **Propozycja naprawy:** Przenieść do `config.js` lub dedykowanego `env.js`.

### SEC-011: Brak rate limiting na endpointach API
- **Risk:** niski (DoS)
- **Lokalizacja:** `server/index.js` (wszystkie app.post)
- **Opis:** Brak rate limiting na `/api/ai/train`, `/api/ai/predict`, `/api/ai/params`. Atakujący może wysyłać tysiące requestów/s.
- **Propozycja naprawy:** Dodać `express-rate-limit` middleware.

### SEC-012: Logowanie pełnych parametrów w setParams
- **Risk:** niski
- **Lokalizacja:** `server/index.js:411`
- **Opis:** `console.log('[WS] setParams from ${socket.id}:', newParams)` — loguje socket ID i pełne parametry. Socket ID nie jest krytyczny, ale razem z parametrami tworzy ścieżkę korelacji.
- **Propozycja naprawy:** `console.log('[WS] setParams received')` — bez wartości.

## Podsumowanie

| Ryzyko | Liczba |
|--------|--------|
| Wysoki | 0 |
| Średni | 2 |
| Niski | 10 |

**Kluczowe wnioski:**
1. Endpointy API zwracają więcej danych niż potrzeba (model internals, training metrics)
2. WebSocket broadcastuje dane do wszystkich — OK dla gry, ale metryki treningowe powinny być prywatne
3. Brak podstawowych zabezpieczeń HTTP (helmet, rate limit, body size limit)
4. Error logging naraża stack trace i ścieżki plików
5. CORS jest skonfigurowany poprawnie (localhost default), ale brak walidacji env var

**Pozytywy:**
- CORS nie jest `*` domyślnie
- WebSocket `socket.emit` używany dla per-klient danych (legal moves, errors)
- Input validation na parametrach (epsilon, networkSize, side) istnieje
- MODEL_DIR i BUFFER_FILE używają `path.join` — bez path traversal

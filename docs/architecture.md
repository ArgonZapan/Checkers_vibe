## Komponenty

### C++ Engine (port 8080)
- Silnik gry (bitboard, reguЕӮy, generowanie ruchГіw)
- REST API вҖ” endpointy do gry i stanu
- NIE robi szkolenia вҖ” tylko logika gry

### Node.js Server (port 3000)
- Express вҖ” serwuje statyczne pliki React
- Proxy вҖ” przekazuje requesty do C++ API
- TensorFlow.js вҖ” sieДҮ neuronowa, szkolenie, predykcja
- Replay buffer вҖ” przechowywanie danych treningowych
- WebSocket вҖ” real-time updates do frontendu

### React Frontend
- Plansza warcab
- Kontrolki gry (tryb, start, stop)
- Dashboard AI vs AI (wykres loss, statystyki, historia)
- Panel parametrГіw (epsilon, rozmiar sieci, restart)

## PrzepЕӮyw danych

### Gra PvAI
1. UЕјytkownik klika ruch вҶ’ React вҶ’ Node.js
2. Node.js вҶ’ C++ API: `POST /api/move`
3. C++ sprawdza legalnoЕӣДҮ, wykonuje ruch, zwraca stan
4. Node.js вҶ’ C++ API: `GET /api/legal-moves`
5. Node.js вҶ’ TensorFlow.js: predykcja ruchu AI
6. AI ruch вҶ’ C++ API: `POST /api/move`
7. Node.js вҶ’ WebSocket вҶ’ React: aktualizacja planszy

### Szkolenie (AI vs AI)
1. Node.js вҶ’ C++ API: pДҷtla gry (ruch po ruchu)
2. KaЕјdy ruch вҶ’ zapis do replay buffer
3. Po grze вҶ’ TensorFlow.js: szkolenie na mini-batch
4. Aktualizacja wag вҶ’ zapis na dysk
5. Kolejna gra automatycznie
